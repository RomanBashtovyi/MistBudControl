# Як додати новий проект

## Крок 1: Створити папку проекту

Створіть папку з унікальним ID проекту (англійською, малими літерами, через дефіс):

```
data/projects/[project-id]/
```

Приклад: `data/projects/warehouse-kyiv-2024/`

**Всі файли проекту (manifest.json та зображення) зберігаються в одній папці!**

## Крок 2: Створити manifest.json

У папці проекту створіть файл `manifest.json` з наступною структурою:

```json
{
  "id": "warehouse-kyiv-2024",
  "title": "Назва проекту українською",
  "description": "Детальний опис проекту. Що було зроблено, які особливості, технології тощо.",
  "category": "warehouse",
  "area": "5000 м²",
  "year": "2024",
  "location": "Київська обл.",
  "thumbnail": "thumbnail.jpg",
  "images": ["01.jpg", "02.jpg", "03.jpg"],
  "featured": true
}
```

### Поля manifest.json:

- **id** (обов'язково) - унікальний ідентифікатор, такий же як назва папки
- **title** (обов'язково) - назва проекту українською
- **description** (обов'язково) - опис проекту
- **category** (обов'язково) - категорія проекту, одна з:
  - `warehouse` - склад
  - `production` - виробництво
  - `logistics` - логістика
  - `metalworking` - металообробка
  - `grain-processing` - переробка зерна
  - `transport-service` - транспортне обслуговування
  - `other` - інше
- **area** (опціонально) - площа проекту
- **year** (опціонально) - рік завершення
- **location** (опціонально) - місцезнаходження
- **thumbnail** (обов'язково) - назва файлу головного зображення (thumbnail.jpg)
- **images** (обов'язково) - масив назв файлів зображень проекту
- **featured** (опціонально) - чи показувати проект на головній сторінці (true/false)

## Крок 3: Додати зображення

**Варіант 1: Локальні файли (рекомендовано)**

Додайте зображення **в ту ж саму папку** `data/projects/[project-id]/`:

- `thumbnail.jpg` - головне зображення (рекомендований розмір: 800x600px)
- `01.jpg`, `02.jpg`, ... - додаткові фото (рекомендований розмір: 1200x800px)

У manifest.json вказуйте лише назви файлів:

```json
"thumbnail": "thumbnail.jpg",
"images": ["01.jpg", "02.jpg"]
```

**Варіант 2: Зовнішні URL (для заглушок)**

Можна використовувати прямі посилання на зображення:

```json
"thumbnail": "https://dummyimage.com/800x600/e5e7eb/6b7280&text=Назва+проекту",
"images": [
  "https://dummyimage.com/1200x800/f3f4f6/6b7280&text=Фото+1"
]
```

Формати: JPG, PNG, WebP

## Крок 4: Перевірка

Проект автоматично з'явиться на сайті після перезавантаження. Перевірте:

- Головну сторінку (якщо `featured: true`)
- Сторінку `/projects` - всі проекти

## Приклад структури:

**З локальними файлами:**

```
data/projects/warehouse-kyiv-2024/
├── manifest.json
├── thumbnail.jpg
├── 01.jpg
├── 02.jpg
└── 03.jpg
```

**З зовнішніми URL (для заглушок):**

```
data/projects/warehouse-kyiv-2024/
└── manifest.json  (містить URL на зображення)
```

**Все в одній папці!** Зручно додавати, редагувати та видаляти проекти.

## Поради:

1. Використовуйте оптимізовані зображення (WebP рекомендується)
2. ID проекту має бути унікальним
3. Для featured проектів використовуйте якісні зображення
4. Опис має бути інформативним та коротким
